[2026-03-09] — Hub / GLPI — Módulos Satélite Invisíveis após Migração de Perfis para Grupos

Status: ✅ Causa raiz confirmada | 🔧 Correção pendente de validação dos IDs

1. Título

Hub / GLPI — Módulos satélite invisíveis por ausência de grupos Hub-App-* no GLPI físico

2. Causa Raiz Confirmada

Os grupos Hub-App-* não existem na instância GLPI física. A função resolve_app_access (auth_service.py:L22-43) busca grupos com prefixo Hub-App- via API REST do GLPI e retorna [] quando nenhum é encontrado. Com app_access = [], o frontend (resolveMenuItems em context-registry.ts:L116-141) filtra e oculta todo módulo que tenha requireApp definido.

3. Solução Aplicada

Arquitetura implementada (dual-path):

Mecanismo Resolve Fonte Arquivo
resolve_hub_roles Papel funcional (tecnico, gestor, solicitante) profile_map + group_map em contexts.yaml auth_service.py:L64-122
resolve_app_access Quais módulos satélite o usuário pode ver Grupos Hub-App-* no GLPI via API auth_service.py:L22-43

Código do extrator (auth_service.py):

async def resolve_app_access(client, user_id: int) -> List[str]:
    group_links = await client.get_sub_items("User", user_id, "Group_User")
    app_access = []
    for gl in group_links:
        gid = gl.get("groups_id")
        group = await client.get_item("Group", gid)
        name = group.get("name", "")
        if name.startswith("Hub-App-"):
            app_access.append(name.replace("Hub-App-", "").lower())
    return app_access

Frontend bloqueador (context-registry.ts):

// Em resolveMenuItems():
if (feature.requireApp && !appAccess.includes(feature.requireApp)) {
  return false; // Bloqueia imediatamente
}

4. Fundamento Técnico

Perfis e grupos no GLPI são entidades independentes com tabelas relacionais distintas (glpi_profiles_users vs glpi_groups_users). O Hub deliberadamente mantém dois caminhos paralelos: perfis definem o "quem você é" (role) e grupos Hub-App-* definem o "o que você pode acessar" (capability). Sem os grupos criados no GLPI, o segundo caminho retorna vazio e o frontend cumpre a trava de segurança projetada.

5. Como Detectar Mais Cedo

  • Usar o endpoint de diagnóstico embutido: GET /api/v1/{context}/auth/diagnose-access?username=X
  • Verificar campo app_access na resposta de POST /auth/login (DevTools > Network)
  • Incluir log de nível INFO em resolve_app_access quando retornar lista vazia

6. Como Prevenir Recorrência

  • Documentar no onboarding que grupos Hub-App-* são pré-requisito para visibilidade de módulos
  • Criar checklist de deploy: "Grupos GLPI criados?" antes de liberar feature com requireApp
  • Considerar fallback visual: módulo aparece bloqueado com tooltip "Solicite acesso ao administrador" em vez de sumir completamente
